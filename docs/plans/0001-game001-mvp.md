## 배경 / 목적

모바일·웹 양쪽에서 플레이 가능한 **찾기 + 상황 해결형 퍼즐** 게임. 참고 레퍼런스(스타일 목표):
Google Play "Brain Puzzle" 류 — JoyGame Studio 계열의 **2D 일러스트 장면 기반** 게임.

레퍼런스 핵심 특징 (스크린샷 분석, 2026-06-27):
- 화면을 꽉 채우는 **2D 일러스트 장면**(방·인물, 카툰/애니풍).
- 레벨 = 하나의 상황 장면 + 하단 **목표 문구** (예: "엄마가 돌아오셨어요! 빨리 방을 치워",
  "잠재적 위기를 해결하기", "도둑맞은 삶을 되찾아라").
- **찾기/상황 해결형** 인터랙션: 장면 속 오브젝트를 탭·드래그해 목표 달성. 함정·발상의 전환 포함.
- 즉 **AI 2D 아트가 게임의 중심 자산** → 이미지 생성 파이프라인이 핵심 인프라.

> 이름: 정식명 확정 전까지 **game001** (가제). 이전 가명칭은 코드·문서·UI 어디에도 사용하지 않는다.

사용자 확정 사항:
- 기술 스택: **웹 우선 + PWA** (Vite + React + TypeScript). (2026-06-27)
- 범위: 플레이 가능한 MVP. (2026-06-27)
- AI 이미지 파이프라인: 같이 구축 (풍경·캐릭터·장면 원화). (2026-06-27)
- 스타일 목표: **장면 기반 찾기·퍼즐** (레퍼런스 이미지). (2026-06-27)

## 현재 상태 / 방향 전환

1차로 **이모지 기반 트릭 레벨 6종**(Stroop·실제크기·숨은세기 등) + 엔진/화면/PWA/배포를 구축해
파이프라인(빌드→Pages 배포)을 검증했다. 이는 **기술 골격 프로토타입**이며, 레퍼런스의
장면 기반 비주얼과는 거리가 있다. 다음 단계에서 **장면(Scene) 기반 레벨 시스템**으로 전환한다.

## 설계 (목표 아키텍처)

### Scene 레벨 모델
- 레벨 = `{ id, goalText, sceneImage, hotspots[], winCondition }`.
- **hotspot**: 장면 위 좌표 영역(상대 %) + 동작(tap/drag/toggle) + 정답 여부/상태.
- 공통 `SceneLevel` 컴포넌트가 배경 이미지 위에 hotspot 레이어를 절대배치 → 레벨 데이터만 추가하면 신규 레벨.
- 기존 임의-컴포넌트 레벨(트릭형)은 "custom" 레벨로 공존 가능(레지스트리에 Component 또는 SceneSpec).

### 아트 파이프라인 연동
- 장면/캐릭터 원화는 `tools/image-gen` 으로 생성 → 선별 후 `public/assets/scenes/`, `public/assets/chars/`.
- hotspot 좌표는 생성 이미지 기준으로 잡는다(에디터/수기). 좌표 보정용 디버그 오버레이 고려.

### 유지 (1차에서 완료)
- Vite+React+TS, vite-plugin-pwa, localStorage 진행도, Home/Game/Win + HUD, GitHub Pages 자동 배포.
